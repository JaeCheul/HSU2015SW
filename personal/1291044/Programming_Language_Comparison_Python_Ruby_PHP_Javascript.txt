1. Python

  문법 자체가 복잡하지 않고 간단하다. 
  변수의 자료형을 선언하지 않고 값만 넣어도 구분을 할 수 있다. 함수나 조건문 등의 유사성도 높아 쉽게 익힐 수 있다. 
  언어 자체가 직관적인 만큼 미국 대학에서는 Python을 우리나라 대학의 C처럼 기초과목 언어로 배우는 추세라고 한다. 
  그만큼 초보자도 배우기 쉬운 언어이다. 또한 다양한 라이브러리를 지원해서 개발 속도를 줄일 수 있다. 
  다른 프로그래밍 언어와 다르게 ';' 와 '{ }'로 구문을 구별하지 않고 들여쓰기로 구분한다. 
  짧은 코드에서는 코딩 속도가 빨라지는 장점이 있지만 코드가 길어질수록 보기가 어려워진다는 문제도 있다. 
  아이러니하지만 이러한 문제를 해결하기 위해 코드를 짧게 짜는 노력을 하다보면 프로그래밍 실력이 느는 장점도 있다.


2. Ruby

  Ruby를 처음 사용해 보았을 때 느낀 점이 Python과 유사하다는 것이다. 변수에 대한 자료형도 필요 없고,
  긴 문장 끝에 ';'을 찍을 필요도 없다. 그래서 Python 공부를 한 후 Ruby 공부를 할 때 문법이 올바른지 확인해 가며 코딩했다.
  차이점이 있다면 함수나 조건문 등에서 끝을 end로 구분할 수 있어, 들여쓰기로 구분하는 Python보다 코드를 쉽게 파악할 수 있다. 
  또한 조건문이나 루프에서 다양한 방법을 제공하여 상황에 맞게 골라서 사용할 수 있다. 예를 들면 한 행에 여러 문장을
  작성할 경우에는 ';'을 사용해서 구분을 해야한다. 더 조사를 해보니 Ruby는 모든 것이 객체인 언어이고, 
  Python과 더불어 스크립트 언어이다. 이러한 특징 때문에 초보자가 접근하기 쉬운 언어이기도 하다. 
  MAC OSX에서는 기본적으로 Ruby가 내장되어 있는만큼 애플에서 밀어주고 있다.


3. PHP

  앞의 두 언어 Python, Ruby와 다르게 웹 전용 개발언어이고, 객체지향적인 언어가 아니다. 
  HTML 파인 안에 '<?php' 와 '?>' 태그를 통해 HTML모드가 아닌 PHP모드로 구분을 하고 그 안에 코드를 작성한다. 
  'echo'를 사용하여 문장이나 변수를 출력하고 요청하는 방법이 간단하다. 또한 PHP는 C나 Java 등 기존에 배운 문법과 유사하다.
  ';'으로 문장을 끝내고 함수와 같은 구문도 '{ }'로 구분한다. 변수 앞에는 '$'을 붙이고 자료형을 명시하지 않다도 되는 등 
  다른점도 있다. 아직 사용해 본적은 없지만 데이터베이스와 연동하여 서버로서의 역할로 자주 쓰인다. 
  그만큼 여러 방면에서 사용할 수 있고 다양한 내장 라이브러리를 지원하며 호환성이 좋다.


4. Javascript

  모든 변수의 자료형 및 함수 선언은 var로 통일되어 있어 쉽게 사용하고 익힐 수 있다. 
  C나 Java와는 다르게 'print'가 아니라 'console.log', 'alert' 등 다양한 방법으로 출력을 하며, 
  PHP와는 다르게 객체기반 언어로 되어있어 보다 보안에 용이하다. 프로젝트를 진행하면서 Javascript로 된 서버코드를 
  작성해 본 경험이 있는데, 이 때 데이터베이스와 연결도 쉽고 작성이 간단하다는 것을 느꼈다. 
  Javascript에 대한 정보가 부족해 자료를 좀 더 조사해 보았다. 
  Javascript는 컴파일, 링크 과정이 필요가 없어 즉시 실행이 가능하고 디버깅도 간편하다. 
  또한 스크립트 언어로 웹브라우저와 같은 환경 내에서 실행되기 때문에, 파일이나 메모리 관리 등이 필요 없다.
