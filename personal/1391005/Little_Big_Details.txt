#과제 : http://littlebigdetails.com 사이트를 보고 10개 이상 맘에 드는 것/인상적인 것들을 뽑고 이유 적기

#맘에드는 기능
1. dropbox : 웹페이지가 깔끔하고 사용하기 편한 점이 맘에 든다. 페이스북 이나 트위터 등등의 웹사이트는 웹페이지에 접속하면 가입창이 굉장히 크게 보이고, 로그인하는 부분은 작거나, 로그인하기 위해서 화면 구석에 있는 버튼을 클릭해야 한다. 이 경우는 사용자가 웹 페이지에 접속했을 때 가입버튼, 로그인 버튼만 있기 때문에 사용자가 기능을 확인하기 쉬운 것 같다. 그리고 웹페이지를 접속했을 때 가입하는 부분이 바로 있는 디자인은 화면이 지저분해서보여 별로 좋아하지 않았는데, Dropbox의 디자인은 깔끔해서 맘에든다

2. Asos : 생각해보면 별로 쓸데 없을 것 같은 사소한 기능인데, 사용자의 기분을 조금 업 시켜줄 수 있는 것 같다. 생일이 오래 남은 사람은 별다른 느낌이 없을 것 같은데, 생일이 다가왔다면 숫자가 작은 것을 보면 괜히 기분이 좋아질 것 같다.

3. Mailchimp : 이것은 기능보다는 디자인에 초점을 둔 것 같다. SEND를 누르기 전까지는 버튼을 누를까 말까 고민하는 듯한 애니메이션이 나오고, SEND를 누를 면 애니메이션도 동일하게 버튼을 클릭한다. 디자인이 너무 귀여워서 맘에든다. 그리고 누를까 말까 하는 애니메이션에서 땀이 흐르는 것이 꼭 고민하는 사람을 보여주는 것 같아서 좋다.

4. Wistia : 이 기능이 없었으면 멤버를 소개하는 페이지가 굉장히 밋밋하고 재미없었을 것 같다. 클릭한 멤버들이 춤을 추면서 좀 더 재미있는 웹페이지를 만들어 주는 것 같다. 

5. MyFitnessPal : 보지 못한 알림을 다시 상기시켜 주는 기능인데, 나처럼 적어놓지 않으면 맨날 까먹는 사람한테는 아주 좋은 기능인 것 같아서 맘에든다. 

6. Sunrise Calendar : 문맥의 타이틀 글자에 따라서 아이콘이 알맞게 변경되는 기능이다. 일단 아이콘이랑 색 조합이 맘에 든다. 그리고 굉장히 똑똑한 기능인 것 같다. 글자를 인식해서 아이콘을 바꾼다니 완전 신기한 기능인 것 같다. 엄청나게 많은 단어와 아이콘이 데이터베이스에 저장되어 있을 것 같다. 개발자가 데이터베이스에 넣느라 힘들었을 것 같다. 이 기능을 보면서 만약에 오타가 생겼을 때는 어떻게 처리하는 지 궁금증이 생겼다. 

7. InVision : 웹 브라우저에서 다른 탭을 클릭했을 때 메시지가 "Don't forget to read this"로 바뀐다. 기능이 되게 귀여운 것 같다.  되게 사소한 기능이고, 말해주지 않으면 잘 몰랐을 것 같은 기능이지만, 참신한 아이디어인 것 같다. 사용자가 호기심에 한번쯤은 다시 웹사이트를 들여다볼 것 같다. 실제로 이렇게 기능하는지 확인해보고 싶어서 웹 사이트를 방문했는데, 웹페이지도 반응형으로 되어있고 디자인도 깔끔해서 너무 맘에 들었다. 나중에 이런 웹페이지를 하나 만들어 보고 싶다.

8. Github : 할로윈 동안 'Spooky'한 컬러로 Contribution 이 바뀌는 기능이다. 그래서 비교를 해봤는데, 원래 Github에서의 컬러는 초록색인데, 할로윈 기간 동안은 주황계열의 색깔로 바뀐다. 할로윈 말고도 다른 특별한 날에도 특정 색깔을 적용할 수 있는 기능이 있으면 더 좋을 것 같다는 생각이다.

9. PinkNews : 뉴스에서 기사 본문을 드래그 할 경우, 단락별로 빨주노초파남보의 무지개 색깔이 나타나는 기능이다. 평소에 뉴스볼 때 드래그 하는 습관이 있는데, 무지개 색깔이 (파스텔톤으로) 나오면 더 예뻐서 좋은 것 같다.

10. Google(weather) : 날씨 정보에서 바람의 세기를 나타낼 때, 바람이 세면 더 두껍고 큰 화살표로, 약하면 얇고 작은 화살표로 표시하는 기능이다. 숫자로 5m/s, 3m/s 이렇게 표시하는 것 보다 한눈에 바람의 세기와 방향을 알 수 있어서 사용자가 봤을 때 더 편하게 날씨 정보를 확인할 수 있는 것 같다.
