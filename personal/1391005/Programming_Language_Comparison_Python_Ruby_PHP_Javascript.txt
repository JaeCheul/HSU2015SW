JavaScript
- 자바와 문법이 비슷하게 느껴져서 처음 시작할 때 덜 어색함을 느꼈습니다. for문과 while문 if else 문을 비슷하게 사용할 수 있어서 문법을 이해하는 데 그다지 어렵지 않았던 것 같습니다.
- 변수를 선언하는데 있어서 var을 사용하면 되었기 때문에 편리했습니다. 자바나 C 같은 경우는 int, double 등 변수의 자료형을 선언해야 했었기 때문에 자료형이 맞지 않으면 에러가 나는 경우가 있었는데, 자바스크립트는 var로 통일되었기 때문에 사용하기 편하게 느껴졌습니다.
- 배열의 요소에 자료 형이 상관이 없다는 점이 좋게 느껴졌습니다. 배열을 선언할 때, int 형이면 int, double 형이면 double 등 자료형에 맞게 배열에 값을 주었어야 했는데, 자바스크립트는 배열 안에 자료 형에 관계없이 넣어도 되던 점이 좋았습니다.
- 자바스크립트는 ==, === 모두를 사용하여 비교를 할 수 있는 점이 좋은 것 같습니다.
- 자바스크립트는 출력할 때 한가지 방법이 아니라 console.log, prompt 등을 사용할 수 있어서 좋았습니다.

Python
- 첫 번째로 파이썬은 코드 해석이 쉬운 점이 장점이라고 생각합니다. 자바에서는 System.out.println을 써서 출력해야 했다면, 파이썬은 print 하나로 출력을 할 수있다는 점, 배열도 [ ]를 써서 간단하게 초기화 할 수 있다는 점이 장점으로 느껴졌습니다.
- 두 번째로 파이썬은 변수를 선언하는데에 있어서 자유롭다는 점이 흥미로웠습니다. 자바스크립트는 자료형을 명시하지 않더라도 var을 써서 변수를 선언했지만, 파이썬은 그렇지 않아도 되었기 때문에 더 쉽게 느껴졌습니다.
- 세 번째로 import문을 쓸 수 있다는 것이 장점으로 생각되어집니다. 코드카데미에서는 날짜와 시간을 import문과 from을 이용하여 작성하였는데, 자바처럼 import를 쓸 수 있어서 새롭게 느껴졌습니다.
- 네 번째로 세미콜론이 아닌 탭으로 문장을 구분하는 것입니다. 코드카데미를 하면서 탭을 주지 않아서 오류가 난 적이 많아 작성하는 당시에는 장점이라고 느껴지지 않았지만, 긴 코드를 작성하거나 프로젝트를 할 경우에는 탭을 주어야 하기 때문에 더 깔끔하게 볼 수 있을 것이라고 생각합니다.
- 다섯 번째로 배열을 다루는데에 편리하다는 점이었습니다. 배열에 있는 요소들을 출력하기 위해 반복문을 돌리지 않고 print를 쓰면 배열의 요소가 출력되었고, 두 배열을 합칠 때도 join을 사용할 수 있어서 좋았습니다.  

Ruby
- 루비는 for, while문과 같은 반복문을 다양한 형태로 출력할 수 있다는 것이 장점으로 느껴졌습니다. until은 조건이 만족할 때 까지, times는 예를 들어 5.times(출력 문장) 이라고 작성하면 5번 출력 되는 등, 반복문을 다양하게 쓸 수 있다는 점과 루프가 끝났을 때 end로 마감을 해주어서 중괄호 보다 훨씬 보기가 편했습니다.
- 루비도 파이썬과 마찬가지로 세미콜론으로 문장의 끝을 명시하지 않는 다는 점이 장점이라고 생각합니다. 
- 루비는 배열에 넣어진 값을 정렬하는데에 장점을 가지고 있는 것 같습니다. C언어의 경우라면 정렬을 하기 위해서 for문을 써야 하지만 루비는 .sort를 쓰면 간단하게 알파벳 순서나 숫자 순서대로 정렬할 수 있었습니다.
- 반복문을 사용했을 때, 중괄호가 아니라 end로 끝나는 것도 루비의 장점이라고 생각합니다.

PHP
- PHP는 자바스크립트 또는 CSS의 연장선이라고 느껴졌습니다. <? php ?>로 php의 시작을 알리는 것이 좋다고 생각합니다.
- 다른 언어와 달리 $변수명을 이용하면 변수를 선언할 수 있어서 변수를 확실히 구분할 수 있었습니다.
- C언어와 비슷하게 느껴졌던 것이 if~else, while문을 비슷한 방식으로 구현했기 때문이었습니다. C언어와 비슷하기 때문에 HTML과 C를 어느정도 다루는 사람이라면 처음에 접근할 때 어렵게 느껴지지는 않을 것 같습니다.
- 다양한 함수를 쓸 수 있다는 것이 장점으로 느껴집니다. 문자열의 길이를 반환할 땐 strlen을 사용할 수 있고, rand를 이용하여 난수를 생성할 수 있는 등 다양한 함수를 쓸 수 있는 것이 좋은 것 같습니다. 
- print와 echo를 이용하여 출력을 다양하게 할 수 있다는 점이 장점으로 느껴집니다.
