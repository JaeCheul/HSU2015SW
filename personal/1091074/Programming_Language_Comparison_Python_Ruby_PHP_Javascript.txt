Python
codeacademy에서 Python을 공부해보니 Python만의 장점을 여러가지 느낄 수 있었는데, 첫번째로 문법 자체가 굉장히 간단한 형태로 구성되어 있다는 점이었다. 따로 변수타입을 지정하지 않아도 되며, char와 string에 큰 차이를 두지 않고 ",'을 사용했다. 조건문을 사용할 때는 tab키를 사용하여 여백을 두고 코드를 작성하여 코드를 한눈에 보기 쉽게 작성할 수 있었다. 두번째로 언어가 초보자들에게도 배우기 쉬울정도로 간결하여 개발 속도가 빠르다는 점이다. 세번째로 다양한 라이브러리가 있다는 점인데, import를 이용하여 쉽게 사용이 가능하다. 네번째로 파이썬은 오픈소스여서 무료로 다운로드하여 사용할 수 있다는 점이다. 또한 컴파일이 느리긴 하나, 인터프리트 방식으로 되어 있어 소스코드를 바로 실행할 수 있도록 지원한다는 점이다.


Ruby
Ruby는 첫번째로 세미콜론이 생략 가능하며, 클래스, 함수, 조건문(if elsif else end, for in end) 뒤에 end형태를 사용하여 끝을 알려주어 코드를 한눈에 보기 편하다. 이러한 문법은 정형화된 서식이 아니기에 사용자가 편하게 사용할 수 있다. 두번째로 모든 것이 객체인 언어라는 것이다.(찾아보니 문법이 자유로워서 사용자의 편의에 따라 절차처럼 사용해도 된다고 한다) 세번째로 Ruby는 스크립트 언어여서 , Python처럼 변수 타입을 지정할 필요 없다는 것이다. 네번째로 Ruby역시 python처럼 인터프리터를 기반의 언어여서 이식성이 높아 어느 운영체제 에서도 사용할 수 있다. 다섯번째로 라이브러리 혹은 내장 클래스가 변경 가능하여 사용자에게 폭넓은 자유도를 부여한다고 한다.  





Java script

php처럼 웹개발에 사용되는 언어이다. Javascript는 Java와 비슷한 언어일 것이라고 생각했는데 문법이 다른 점이 많았다. Java script의 첫번째 장점은.. Python, Ruby의 경우 변수 선언 없이 변수를 사용가능한데, javascript 역시 변수 선언시 var를 사용하거나 별다른 선언 없이 사용 가능하여 편리하다는 것이다. (없이 선언시 전역 변수로 선언된다고 함) 두번째 로 네트워크와 서버쪽에 부담되지 않게 클라이언트 쪽에서 해석 된다는 것이다.  세번째로 javascript에 대한 정보를 찾아보니 객체지향 언어이기 때문에(php도 객체지향언어) 재사용이 가능하며, 이식성이 좋아 플랫폼에 의존하지 않고 지원만 해주는 브라우저라면 어디든 사용할 수 있다고한다. 네번째로 별도로 인터프리터가 필요하지 않고, 웹 브라우저가 Java script를 해석한다. 다섯번째로 javascript를 조사해보니 웹 브라우저의 Script엔진이 직접 코드를 해석하여처리해 적은 시스템 부하와 환경에 독립적인 장점이 있다고 한다.
