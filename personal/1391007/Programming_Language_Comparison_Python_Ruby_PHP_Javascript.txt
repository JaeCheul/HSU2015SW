Python
1.	문법이 간단하여 배우기 쉽다. 간결한 소스로 이루어져 있어 보기도 쉽고 쓰기도 쉽다. 그래서 외국에서는 c언어보다 먼저 배우기도 한다.(추가 조사)
2.	세미콜론을 쓰지 않고 Tab(들여쓰기)를 이용한다. 세미콜론을 쓰지 않아 다른 언어를 하던 사람들은 습관을 고쳐야 한다.
3.	변수 선언이 자유롭다. 변수를 만들면 자동으로 그에 맞는 타입이 설정된다. 자바스크립트에서 var를 만드는 것보다 더 변수 선언에 있어 타입을 신경쓸 필요가 없다.
4.	Lambda함수를 이용해 코드를 줄일 수 있다. 함수를 필요에 따라 일회용으로 만들어서 쓰고 버리는 형태이다.
5.	줄을 꼭 맞춰야한다. 실행되는데 까다로운 것 같다. 
Javascript
1.	함수를 사용할 때 반환과 매개변수 타입 설정으로부터 자유롭다. 
2.	배열을 만들 때 동적 생성한다.
3.	Var로 변수의 타입을 정의한다. 변수를 int float 각 각 쓸 필요 없다. Var를 쓰면 자동으로 지원한다. 초보자들의 실수를 줄일 수 있는 것 같다.
4.	===를 사용한다. 자바스크립트에서 ===를 처음 보았는데 값 외의 타입도 구분할 수 잇다는 점이 다르다.
5.	html안에서 쓸 수도 있고, 따로 자바스크립트파일도 만들 수 있다. 간단한 웹이라면 html안에서 함께 쓸 수 있고, 비교적 복잡하다면 따로 파일을 만들어 관리할 수 있다.
PHP
1.	배열의 반복이 쉽다. foreach문을 사용하여 배열의 반복문을 빠르게 이용할 수 있다.
2.	HTML 문서 내부에 입력할 수 있다. HTML문서안에 <?php (코드) ?>안에 php 문법을 이용하여 웹을 프로그래밍한다. 이는 간편하게 html문서를 만들 수 있게 한다.
3.	수학함수를 쉽게 쓸 수 있다. HTML안에서 수학함수를 이용해 계산을 빠르게 구현할 수 있다.
4.	문법이 간단하다. 루비나 파이썬을 배울 때보다 시간도 적게 걸리고, 문법도 비교적 간단하였다.
5.	c언어와 비슷하다. 문법이나 형태가 비슷해서 c를 배운 사람이라면 쉽게 배울 수 있을 것 같다.
Ruby
1.	세미콜론이 없다. 파이썬과 마찬가지로 세미콜론이 없기 때문에 보기 쉽다.
2.	Block, proc를 사용하는 것은 루비만의 특징이다.
3.	Begin end로 긴 주석을 관리한다. 주석을 보통 // /**/ <--! -- >이런 특수문자로 썼었는데 begin과 end로 쓰니 신기했다.
4.	파이썬과 비슷하다. 세미콜론을 찍지 않는 것, 변수의 타입을 따로 선언할 필요가 없다는 점 등 파이썬을 아는 사람이라면 쉽게 배울 수 있다.
5.	End를 사용하여 끝을 쉽게 알 수 있다. for문에서 {}를 찾아 헤맸다면 루비에서는 end가 반복문이나 조건문의 끝을 알려준다.
